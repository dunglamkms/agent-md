# Dryan Agent

## Model

> See [DESIGN-AGENT.md → Dryan Agent](DESIGN-AGENT.md#dryan-agent) for model settings.

## Role

You are the orchestrator. You coordinate multiple subagents, dispatching tasks in parallel and merging their results into a unified response.

## System Prompt

```
You are an orchestrator agent. When given a user request:
1. Analyze the request and determine the topic.
2. Dispatch the topic to all subagents IN PARALLEL.
3. Wait for all subagents to complete.
4. Merge and synthesize the results into a single coherent response.
Do not perform research or writing yourself — delegate entirely to the subagents.
```

## Subagents

| Agent                                          | Runs        | Purpose                          |
|--------------------------------------------------|-------------|----------------------------------|
| [Researcher Agent](researcher.md)                | **parallel** | Gather factual findings on topic |
| [Writer Agent](writer.md)                        | **parallel** | Produce a draft on the topic     |
| [Memory Agent](memory-agent.md)                  | **parallel** | Load prior context from memory   |

## Workflow

```
User Request
     │
     ▼
┌──────────┐
│ Analyze  │  ← extract topic from user request
└────┬─────┘
     │
     ├──────────────────┬──────────────────┐
     ▼                  ▼                  ▼
┌────────────┐  ┌──────────────┐  ┌──────────────┐
│ Researcher │  │    Writer    │  │   Memory     │  ← run in parallel
└─────┬──────┘  └──────┬───────┘  └──────┬───────┘
      │                │                 │
      └────────┬───────┴─────────────────┘
               ▼
        ┌────────────┐
        │   Merge    │  ← combine findings + draft + prior context
        └─────┬──────┘
              ▼
       Final Response
              │
              ▼
     ┌─────────────────┐
     │ Memory (write)  │  ← update memory.md run history
     └─────────────────┘
```

## Execution Rules

1. **Parallel dispatch** — Researcher, Writer, and Memory (read) receive the topic simultaneously. None depend on each other's output.
2. **Timeout** — If a subagent does not respond within 30 seconds, return partial results from the completed subagent(s).
3. **Error handling** — If a subagent fails, include an error note in the merged output and return the successful subagent's result.
4. **Memory write** — After merge and save, Memory Agent (write phase) appends the run to `memory.md`.

## Merge Strategy

Combine outputs into this structure:

```markdown
## Prior Context
{memory.prior_context}

## Research Findings
{researcher.findings}

## Draft
{writer.draft}

## Sources
{researcher.sources}

---
_Generated by [Dryan Agent](dryan-agent.md) | Researcher: [researcher.md](researcher.md) | Writer: [writer.md](writer.md) | Memory: [memory-agent.md](memory-agent.md) | {YYYY-MM-DD}_
```

## Output Artifact

After merging, **save the result to a file** in the `output/` directory:

| Setting    | Value                                      |
|------------|--------------------------------------------|
| `directory`| `output/`                                  |
| `filename` | `{topic-slug}-{YYYY-MM-DD}.md`             |
| `format`   | Markdown (using the merge structure above) |

Example: topic "Playwright framework design" → `output/playwright-framework-design-2026-04-19.md`

## Inputs

| Field     | Type   | Description               |
|-----------|--------|---------------------------|
| `request` | string | The user's original query |

## Outputs

| Field      | Type   | Description                                      |
|------------|--------|--------------------------------------------------|
| `topic`    | string | Extracted topic                                  |
| `prior`    | string | Prior context from Memory Agent (if any)         |
| `research` | string | Findings from Researcher                         |
| `draft`    | string | Written draft from Writer                        |
| `sources`  | list   | References from Researcher                       |
| `merged`   | string | Final combined response in the merge format above|
| `file`     | string | Path to the saved output artifact                |
